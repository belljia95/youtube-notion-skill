架构总览

三层结构，一键运行

把这个脚本想象成三层结构：最上面是你触发命令的入口；中间是统筹全局的主程序；最底下是两个外部服务，负责真正干活。点击任意模块查看详细说明。

触发层 — 你的操作入口
💬
Claude Code
输入 /youtube-notion 触发一切
📄
SKILL.md
告诉 Claude 该跑哪个脚本、怎么汇报

Claude Code — 指令入口

你在终端输入 /youtube-notion，Claude Code 识别到这是一个已安装的 Skill，加载 SKILL.md 里的指令，然后自动执行后续所有步骤，最后用中文把结果汇报给你。你不需要知道任何技术细节。

SKILL.md — 技能说明书

存放在 ~/.claude/skills/youtube-notion/，相当于一份操作手册。它告诉 Claude：运行哪个 Python 脚本、脚本在哪里、输出结果后怎么整理成可读的中文摘要。

↓
逻辑层 — 总指挥
⚙️
monitor.py
主程序，负责调度所有 API 调用
🔑
config.py
存放 API 密钥，本地私有不上传

monitor.py — 主程序

整个脚本的大脑。执行顺序：① 读取 Notion 博主列表 → ② 对每个频道查询 YouTube 最近 7 天新视频 → ③ 检查视频是否已存在（去重）→ ④ 获取完整描述 + 播放量 + 点赞数 → ⑤ 写入 Notion 选题数据库。

config.py — 密钥文件

存放 YouTube API Key 和 Notion Integration Secret。这个文件被 .gitignore 排除，永远不会上传到 GitHub，只存在你自己的电脑上。其他人克隆仓库后需要自己创建这个文件。

↓
服务层 — 外部 API
▶️
YouTube Data API
提供频道视频列表、完整描述、播放量
📋
Notion API
读取博主列表，写入选题追踪数据库

YouTube Data API v3

脚本调用两个端点：search（获取频道近期视频列表）和 videos（获取完整描述 + 统计数据）。每天免费配额 10,000 单位，监控 4 个频道约消耗 800 单位，完全够用。

Notion API

脚本读取「YouTube博主」数据库获取要监控的频道，然后把新视频写入「YouTube博主选题追踪」数据库。写入字段包括：标题、链接、博主、描述、播放量、点赞数、发布日期。

工具箱

用到的外部服务

这个脚本只依赖两个外部 API，都有免费额度，日常使用完全不花钱。

▶️
YouTube Data API v3

Google 提供的官方接口，用于获取频道视频列表、视频描述、播放量、点赞数等数据。稳定、官方，不需要下载任何软件。

免费额度：10,000 单位/天 本项目消耗：约 800 单位/天（4个频道）
📋
Notion API

Notion 官方接口，支持读取和写入数据库。通过创建 Internal Integration 获取密钥，可以连接你自己的 Notion 工作区。

完全免费 无请求数量限制（合理使用范围内）
